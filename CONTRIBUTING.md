# Contributing to Laravel User Activity Monitor

We love your input! We want to make contributing to Laravel User Activity Monitor as easy and transparent as possible, whether it's:

- Reporting a bug
- Discussing the current state of the code
- Submitting a fix
- Proposing new features
- Becoming a maintainer

## Development Process

We use GitHub to host code, track issues and feature requests, as well as accept pull requests.

## Pull Requests

Pull requests are the best way to propose changes to the codebase. We actively welcome your pull requests:

1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes.
5. Make sure your code lints.
6. Issue that pull request!

## Bug Reports

**Great Bug Reports** tend to have:

- A quick summary and/or background
- Steps to reproduce
  - Be specific!
  - Give sample code if you can
- What you expected would happen
- What actually happens
- Notes (possibly including why you think this might be happening, or stuff you tried that didn't work)

## Development Setup

1. Clone the repository
2. Run `composer install`
3. Run `composer test` to run the test suite

## Testing

- We use PHPUnit for testing
- Please add tests for any new functionality
- Run `composer test` to execute the test suite
- Ensure all tests pass before submitting a PR

## Coding Style

- We follow PSR-12 coding standards
- Use meaningful variable and method names
- Add docblocks to your methods
- Keep methods focused and small
